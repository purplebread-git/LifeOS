# Dogfooding Era

После merge #37 (`lifeos chat`) LifeOS переходит из режима «строим платформу»
в режим «пользуемся продуктом».

Это граница в истории проекта, а не косметика roadmap.

## Эпохи

| Эпоха | PR | Суть |
|---|---|---|
| Ядро | #1–#26 | Построение Core Runtime |
| Платформа | #27–#33 | Доказательство расширения (Plugin API) |
| Streaming + API | #34–#36 | Один ReAct, два транспорта, SSE |
| Продукт / Dogfooding | #37+ | Ежедневное использование как главный тест |

## Критерий успеха

**Было:** «Все тесты зелёные.»

**Стало:** «Я прожил сегодняшний рабочий день внутри LifeOS.»

Тесты остаются обязательными. Они больше не главный источник истины о
качестве системы.

## Правило следующих PR

Каждый PR после #37 отвечает на **один** из двух вопросов:

1. **Что сегодня реально помешало пользоваться?**  
   (UX, ошибки, команды, диалоги, документы, память…)
2. **Что пришлось делать в ChatGPT вместо LifeOS?**  
   («Я всё равно открывал ChatGPT, чтобы…»)

Источник изменений — ежедневная эксплуатация, не абстрактный roadmap.

## Сознательный запрет (ближайшие 2–4 недели)

Не начинать без реальной боли из dogfooding:

* MCP;
* Multi-Agent / Planner / Workflow Engine;
* Auto Plugin Discovery;
* сложную Observability «на будущее».

Не потому что они не нужны — потому что сейчас впервые можно собрать
настоящие требования.

## Когда снова открывать большой Roadmap

Не раньше **2–4 недель** активного использования. Затем по заметкам:

* какие проблемы повторялись;
* какие фичи просились постоянно;
* какие архитектурные ограничения реально встретились.

Только после этого — следующий крупный блок (часто Observability, но
приоритет может оказаться другим).

## Как пользоваться сейчас

```bash
uv run lifeos chat
uv run lifeos chat --conversation-id my-day
```

Сквозной путь: человек → CLI → Agent → ConversationEngine → Tools / Memory /
Knowledge / Plugins → LLM → ответ.
