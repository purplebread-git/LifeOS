Vision

LifeOS — это не просто чат с LLM.

Цель проекта — создать модульную агентную платформу, способную развиваться от простого диалогового агента до полноценной операционной системы для цифрового помощника.

Ключевая идея:

* агент не должен зависеть от конкретной LLM;
* агент не должен зависеть от конкретных инструментов;
* агент не должен зависеть от конкретного хранилища памяти;
* все внешние возможности подключаются через абстракции и DI.

Любой компонент должен иметь возможность быть заменённым без изменения бизнес-логики системы.

⸻

Architectural Principles

Dependency Inversion

Высокоуровневая логика не зависит от реализаций.

Используем:

* ABC
* Protocol (при необходимости)
* dependency-injector

Запрещено:

* создавать инфраструктурные объекты внутри бизнес-логики;
* импортировать конкретные реализации в доменный слой.

⸻

Composition Root

Все реальные реализации собираются только в DI Container.

Контейнер является единственной точкой сборки приложения.

⸻

Tool-Driven Agent

Агент не знает о конкретных инструментах.

Инструменты являются независимыми capability-модулями.

Агент получает список доступных инструментов через ToolManager.

⸻

Memory as Capability

Память не является частью агента.

Память представляет собой отдельную подсистему через MemoryProvider.

Агент может:

* сохранять воспоминания;
* искать воспоминания;
* использовать воспоминания при построении контекста.

Но агент не зависит от конкретной реализации памяти.

⸻

Provider-Based Infrastructure

Все внешние интеграции должны скрываться за провайдерами.

Примеры:

* LLMProvider
* MemoryProvider
* ConversationRepository

Это позволяет менять инфраструктуру без изменения бизнес-логики.

⸻
